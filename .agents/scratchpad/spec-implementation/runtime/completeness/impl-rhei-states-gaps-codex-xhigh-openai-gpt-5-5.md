# Completeness Audit: `docs/functional-spec/rhei-states.spec.md`

**Auditor:** codex[xhigh]:openai:gpt-5.5
**Spec source:** `runtime/manifests/impl-rhei-states-spec.txt` -> `docs/functional-spec/rhei-states.spec.md`
**Scope:** implementation in repository root `/home/vjovanov/c/rhei`
**Method:** enumerate normative claims in the assigned spec and classify each as `covered`, `partial`, `missing`, or `not-normative`. Code quality is not assessed.

## Summary

The implementation covers the new state schema shape, target selector parsing, most structural validation, program states, artifact entry/exit checks, template substitution, profile/node-policy validation, reset-to-profile-initial behavior, default states, and most `rhei run` gating semantics.

Material completeness gaps:

- `rhei next` still treats `StateDef.initial` as the claim/auto-transition source instead of resolved profile initial states.
- `rhei complete` does not block completion from gating states, including default `human-review`.
- Poll transition condition names `pollAttempts` and `pollMaxAttempts` are not implemented.
- `visits` / `poll` counters are keyed only by state, not by fanout target/model.
- Empty `all_targets: []` cannot be distinguished from omission and is not rejected.
- Top-level `models` and state tooling registry ids are not fully validated against merged settings at `rhei validate` time.
- MCP availability is registry/support based, not actual server start/handshake based.
- `{model.provider}` / `{model.name}` template and artifact resolution is incomplete for legacy model profiles.
