## Summary

Covered areas:

- Shared snapshot reference parsing for `snapshot show` and `snapshot continue`.
- `rhei snapshot list` CLI shape, text/JSON rendering, basic filters, and produced-by defaults.
- `rhei snapshot show` manifest rendering and transcript preview.
- `rhei snapshot gc` CLI shape, policy filtering, dry-run, deletion, generation grouping, operator exclusion by default, and live-run interlock checks.
- `snapshots` settings parsing/merge and configurable cache root.
- Minimal validator structs for `snapshot.emit` and `snapshot.inherit`.

Main partial or missing areas:

- Child task traversal is missing in orphan detection and active-inherit GC protection.
- `gc --older-than` combined with `--keep-generations` applies the age filter before retention grouping, so retention is not evaluated against all generations in an identity.
- `rhei snapshot continue` is a precondition-checking stub; interactive spawn, transcript capture, operator manifest writing, lock acquisition, no-capture behavior, and atomic sibling generation allocation are missing.
- `rhei run --from-snapshot` is mostly a CLI surface plus missing-`snapshot.inherit` guard; actual reference resolution, constraint checks, compatibility checks, ambiguity reporting, and preload override behavior are missing.
- Redactor execution and the full privacy/process contract are missing.
- Runtime snapshot writes, auto-emit, adapter-specific end-to-end support, `cache_beneficial` TTL use, and full snapshot grammar validation are missing.
