# Quality Review pass 2 - target codex[xhigh]:openai:gpt-5.5

- Q-profile-allowed-not-enforced-on-transition: Transitions can move a node into a state excluded by its resolved profile
  - Severity: medium
  - File: crates/rhei-cli/src/cli/system_transition_execution.rs
  - Detail: `node_policy` profiles define the authoritative `allowed` state set for each node, but the transition commit path only checks that `from`/`to` are globally valid states and that a global transition rule exists. It never resolves the task's kind/level profile before writing the new `**State:**`, and callback redirects have the same gap. A narrowed profile such as `fast-track: allowed: [pending, completed]` can still be moved through a globally declared `pending -> review` edge by `rhei transition`, `rhei next`, or `rhei run`; the resulting plan is invalid under the validator's own profile check.
  - Evidence: The spec says the resolved profile's `allowed` set is "the authoritative list of states that node may ever hold" and authored node states must appear in that set (`docs/functional-spec/rhei-states.spec.md:820-842`). Static validation enforces this for already-authored states in `validate_task_state_against_profile` (`crates/rhei-validator/src/validator/validator_entry.rs:248-275`). At runtime, `execute_transition_with_origin` validates only `machine.is_valid_state(to)` and a matching global rule before committing (`crates/rhei-cli/src/cli/system_transition_execution.rs:18-26`, `:91-107`, `:279-328`), while callback redirects repeat the same global-only check (`:245-277`). Automatic selection returns the first globally applicable transition without checking whether `rule.to` is in the task's resolved profile (`crates/rhei-cli/src/cli/ready_transition.rs:254-302`).
  - Suggested fix: Before selecting or committing any transition, resolve the task node from the parsed plan, compute its profile with `profile_for_node(task.kind, task.id.depth())`, and reject/filter any destination not in `profile.allowed`; apply the same check after callback redirect resolution and add coverage for a profile that excludes an otherwise globally valid transition target.
